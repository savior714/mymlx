# Stack Upgrade Exceptions Policy

## 목적

스택 업그레이드 강제 정책 적용 시 발생하는 예외를 기한 기반으로 관리한다.

## 관리 원칙

- 예외는 임시 상태이며 영구 허용이 아니다.
- 각 예외는 아래 필드를 반드시 포함한다.
  - 비허용 기술명
  - 유지 이유
  - 제거 목표일
  - 대체 진행률
  - 다음 재검토일
  - 오너

## 운영 규칙

- 예외 승인 없이 비허용 스택을 신규 도입하는 변경은 차단한다.
- 재검토일마다 예외 유지/축소/제거를 재결정한다.
- 목표일 만료 예외는 연장 사유를 갱신하지 않으면 종료 대상으로 처리한다.

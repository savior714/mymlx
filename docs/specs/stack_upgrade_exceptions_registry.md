# Stack Upgrade Exceptions Registry

## 목적

`docs/specs/stack_upgrade_enforcement_guide.md`의 강제 규칙과 현재 프로젝트 아키텍처 간 충돌을 **기한 있는 예외**로 관리한다.

예외는 임시 상태이며, 반드시 제거 목표일과 재검토일을 포함해야 한다.

---

## 예외 필드 규격 (필수)

- 비허용 기술명
- 유지 이유
- 제거 목표일
- 대체 진행률
- 다음 재검토일
- 오너

---

## 예외 목록

### EX-001: Starlette 유지 (FastAPI 미전환)

- 비허용 기술명: `Starlette` (가이드상 `FastAPI` 요구)
- 유지 이유: 현재 서버는 OpenAI 호환 프록시 중심이며 Starlette 기반으로 안정 동작 중. 즉시 전환 시 라우팅/미들웨어 회귀 위험이 큼.
- 제거 목표일: `2026-07-31`
- 대체 진행률: `0%` (타당성 검토 전)
- 다음 재검토일: `2026-05-31`
- 오너: `backend`

### EX-002: SQLModel 비적용

- 비허용 기술명: `No SQLModel` (가이드상 `SQLModel` 요구)
- 유지 이유: 현재 핵심 요구사항이 DB 영속 계층이 아니며, 불필요 도입 시 복잡도와 유지비만 증가.
- 제거 목표일: `DB 기능 스펙 승인 시점`
- 대체 진행률: `N/A`
- 다음 재검토일: `2026-06-30`
- 오너: `architecture`

### EX-003: pyx 비적용 (`uv run` 유지)

- 비허용 기술명: `No pyx` (가이드상 실행/스크립팅 `pyx` 요구)
- 유지 이유: 현재 `uv run` 기반 개발/운영 경로가 정착되어 있고 전환 실익이 검증되지 않음.
- 제거 목표일: `2026-08-31`
- 대체 진행률: `0%`
- 다음 재검토일: `2026-06-15`
- 오너: `developer-experience`

### EX-004: Bash 스크립트 유지 (Nushell 미전환)

- 비허용 기술명: `bash` (가이드상 `Nushell` 요구)
- 유지 이유: `run.sh`, `run_models.sh`, `verify.sh` 등 핵심 운영 경로가 bash에 의존. 전면 전환은 별도 PoC/회귀 검증 필요.
- 제거 목표일: `2026-09-30`
- 대체 진행률: `10%` (전환 계획 수립 완료)
- 다음 재검토일: `2026-06-30`
- 오너: `platform`

## 운영 규칙

1. 예외 없이 비허용 스택을 신규 도입하는 PR은 차단한다.
2. 예외 항목은 재검토일마다 유지/축소/제거를 결정한다.
3. 목표일이 지난 예외는 만료로 간주하고, 연장 시 사유를 갱신한다.

# apply_diff 이스케이프 규칙

## 개요

`apply_diff` 도구는 Diff 기반 파일 수정을 수행하지만, **Diff 구분자 마커**가 내용에 포함되어 있을 경우 파싱 오류가 발생할 수 있습니다. 이 문서는 이를 방지하기 위한 이스케이프 규칙을 정의합니다.

## 이스케이프 대상 마커

`apply_diff`의 `diff` 파라미터에 포함될 수 있는 다음 마커는 **백슬래시로 이스케이프**해야 합니다:

| 원본 마커 | 이스케이프된 형태 |
|-----------|------------------|
| `<<<<<<<` | `\<<<<<<<` |
| `=======` | `\=======` |
| `>>>>>>>` | `\>>>>>>>` |

## 왜 이스케이프가 필요한가?

`apply_diff` 도구는 내부적으로 diff 구문을 파싱할 때 위 마커들을 구분자로 사용합니다. 만약 내용 자체에 이러한 마커가 포함되어 있다면:

1. **파싱 오류**: 도구가 내용의 마커를 diff 구분자로 오인
2. **수정 실패**: 전체 또는 일부 내용이 올바르게 적용되지 않음
3. **데이터 손상**: 예기치 않은 파일 변경

## 예시

### ❌ 올바르지 않은 예 (이스케이프 없음)

```python
# 이 코드는 오작동합니다
diff = """
<<<<<<< SEARCH
line1
=======
line2
>>>>>>> REPLACE
"""
```

### ✅ 올바른 예 (이스케이프 적용)

```python
# 이 코드는 정상 동작합니다
diff = """
\<<<<<<< SEARCH
line1
\=======
line2
\>>>>>>> REPLACE
"""
```

## 적용 대상

이 규칙은 다음 상황에 적용됩니다:

- **주석**에 `<<<<<<<`, `=======`, `>>>>>>>` 포함 시
- **문자열 리터럴**에 위 마커 포함 시
- **여러 줄 텍스트**에 위 마커가 포함될 가능성이 있을 때

## 자동화 체크리스트

`apply_diff` 사용 전, 다음을 확인하세요:

- [ ] `diff` 파라미터에 `<<<<<<<`가 있는가? → `\<<<<<<<`로 변경
- [ ] `diff` 파라미터에 `=======`가 있는가? → `\=======`로 변경
- [ ] `diff` 파라미터에 `>>>>>>>`가 있는가? → `\>>>>>>>`로 변경

# 구현 계획: 터미널 UI 상태 유지 및 스마트 디폴트 도입

## 1. 개요
현재 `run.sh`는 실행할 때마다 모델 선택과 파라미터(Temperature, Max Tokens 등) 입력을 매번 요구합니다. 이를 개선하여 마지막에 사용한 설정을 기억하고, 사용자가 변경을 원할 때만 입력 창을 띄우도록 하여 편의성을 높입니다.

## 2. 목표
- 마지막으로 성공적으로 실행된 모델 및 옵션값을 로컬 파일(`.last_run`)에 저장.
- 실행 시 "마지막 설정 사용 여부"를 먼저 묻고, 승인 시 즉시 기동.
- 변경을 원할 경우에만 기존의 모델 리스트 및 파라미터 입력 UI 노출.
- 파라미터 입력 시 마지막 사용 값을 기본값(Default)으로 제시.

## 3. 상세 작업 내용

### 3.1. `run.sh` 스크립트 수정
- **상태 로드**: 스크립트 시작 시 `${ROOT}/.last_run` 파일 존재 여부 확인 및 로드.
- **모델 선택 로직 개선**:
    - `LAST_MODEL`이 있으면 "Use last model ($LAST_MODEL)? [Y/n]" 확인.
    - 'No' 선택 시에만 `select` 문을 통한 모델 리스트 노출.
- **파라미터 입력 로직 개선**:
    - `LAST_TEMP`, `LAST_MAX`, `LAST_LOG` 값을 `read -p`의 기본값으로 활용.
    - 예: `read -p "Temperature [$LAST_TEMP]: " USER_TEMP`
- **상태 저장**: 서버 기동 직전에 현재 선택된 값들을 `.last_run` 파일에 저장.

## 4. 검증 계획
- [ ] `run.sh`를 처음 실행했을 때는 평소처럼 모든 값을 묻는지 확인.
- [ ] 실행 종료 후 `.last_run` 파일이 생성되고 값이 올바른지 확인.
- [ ] 재실행 시 "Use last model?" 질문이 나오는지 확인.
- [ ] 'Y' (또는 Enter) 입력 시 파라미터 질문 없이 즉시 서버가 뜨는지 확인.
- [ ] 'n' 입력 시 모델 선택 리스트가 다시 나오는지 확인.
- [ ] 파라미터 변경 시 새 값이 다음 실행에 반영되는지 확인.

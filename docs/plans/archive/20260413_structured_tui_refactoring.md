# 구현 계획: run.sh 구조화된 TUI 및 메뉴 개선

## 1. 개요
현재 `run.sh`는 선형적인 워크플로우를 가지고 있습니다. 이를 보다 직관적이고 반복 사용이 편리하도록 **구조화된 메뉴 시스템**으로 개편합니다. 사용자는 메인 메뉴에서 모델을 즉시 선택하여 실행하거나, 설정(Options) 메뉴로 들어가 각 파라미터를 개별적으로 수정할 수 있습니다.

## 2. 목표
- **메인 메뉴 구성**:
    1. **Run Model**: 구동 가능한 모델 리스트를 보여주고 선택 시 서버 실행.
    2. **Options**: 서버 설정 및 생성 파라미터를 개별적으로 수정 가능한 서브 메뉴.
    3. **Quit**: 스크립트 종료.
- **옵션 개별 수정**: "Options" 메뉴 내부에서 특정 항목(예: Temperature)만 골라 수정 가능.
- **상태 유지**: 수정된 옵션은 파일에 저장되어 다음 실행 시에도 유지됨.
- **사용자 경험 개선**: 반복적인 입력 없이 엔터만으로 빠른 실행 지원.

## 3. 상세 설계

### 3.1. 메뉴 구조 (TUI)
```text
=======================================
    MLX Server Control Center
=======================================
1. Run Model
2. Options
3. Quit
---------------------------------------
Selection [1-3]: 
```

**[1. Run Model] 선택 시**
- `~/Desktop/models` 폴더 내 모델 리스트 노출 (fzf/rofi 또는 숫자 메뉴).
- 모델 선택 시 즉시 서버 기동.

**[2. Options] 선택 시**
```text
---------------------------------------
    Current Settings
---------------------------------------
1. Host: 127.0.0.1
2. Port: 8080
3. Temperature: 0.7
4. Max Tokens: 1024
5. Top-P / Top-K: ...
6. Adapter Path: ...
7. Log Level: INFO
8. Memory: Prompt Cache=..., Decode=...
9. Go Back
---------------------------------------
Selection [1-9]: 
```

### 3.2. 핵심 함수 (Functions)
- `load_config()` / `save_config()`: 설정 파일(`~/.mlx-server/config`) 로드 및 저장.
- `main_menu_loop()`: 전체 메뉴 루프 처리.
- `menu_run_model()`: 모델 리스트 표시 및 실행.
- `menu_options()`: 설정 요약 표시 및 수정 항목 선택.
- `edit_setting(key, label)`: 특정 설정을 사용자로부터 입력받아 갱신.
- `start_server(model)`: 최종 설정값으로 서버 실행.

### 3.3. 기술적 제약 및 고려사항
- **Bash 3.2 호환성**: macOS 기본 셸 환경에서 동작하도록 `tr`, `printf` 등 표준 도구 사용. (연관 배열 사용 지양 또는 우회)
- **에러 핸들링**: 잘못된 메뉴 번호 입력 시 루프 지속.
- **종료 로직**: 서버 종료 후 다시 메인 메뉴로 복귀할지, 완전 종료할지 사용자 선택 지원.

## 4. 작업 단계
1. `run.sh`의 기존 로직을 함수 단위로 리팩토링 및 모듈화.
2. 메인 메뉴 루프 구현.
3. 옵션 수정 서브 메뉴 구현.
4. 설정 저장/로드 로직과 통합.
5. 최종 테스트 및 가독성(UI) 조정.

## 5. 검증 계획
- [ ] 메인 메뉴에서 1번을 누르면 모델 리스트가 정상적으로 나타나는가?
- [ ] 2번(Options)에서 특정 값을 수정한 뒤, 다시 들어왔을 때 변경된 값이 유지되는가?
- [ ] 설정을 변경하고 모델을 실행했을 때, 변경된 파라미터가 서버 인자로 잘 전달되는가?
- [ ] `Quit` 선택 시 스크립트가 안전하게 종료되는가?
- [ ] 설정 파일에 값이 올바르게 저장/로드되는가?
